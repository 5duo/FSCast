# 青鸪车机蓝牙音频输出问题解决方案

## 问题根源

经过深入调查，发现青鸪车机系统实施了**多层次的音频路由限制**：

### 1. UID限制（核心问题）
- 系统应用（uid=1000）: ✅ 可以输出到蓝牙A2DP
- 第三方应用（uid>10000）: ❌ 只能输出到扬声器

### 2. AudioPolicyManager检查
- `AudioPolicyManager.getOutputForAttrInt()` 方法中有额外的UID/flags检查
- 即使取消A2DP静音，第三方应用仍被强制路由到扬声器

### 3. SELinux限制
- 应用SELinux上下文：`untrusted_app`
- 无法直接执行`su`命令修改系统属性

## 调查结果对比

| 应用 | UID | Flags | A2DP输出 | 说明 |
|------|-----|-------|----------|------|
| 网易云音乐（系统版） | 1000 | 0xA00 | ✅ 蓝牙 | 系统应用 |
| 高德地图 | 10010 | 0x800 | ❌ 扬声器 | 第三方应用 |
| Bilibili | 10068 | 0x800 | ❌ 扬声器 | 第三方应用 |
| 我们的应用 | 10080 | 0xA00 | ❌ 扬声器 | 第三方应用 |

**结论：所有第三方应用都无法输出到A2DP，这是青鸪系统的安全设计。**

## 可能的解决方案

### 方案A：联系车机厂商（推荐）⭐
- **可行性**：高
- **复杂度**：低
- **效果**：永久性解决方案

**操作步骤**：
1. 联系车机厂商的技术支持
2. 说明你的应用需要输出音频到蓝牙耳机
3. 请求将你的应用加入音频白名单
4. 厂商会更新系统配置或提供定制固件

### 方案B：系统签名应用
- **可行性**：中（需要厂商协助）
- **复杂度**：高
- **效果**：永久性解决方案

**操作步骤**：
1. 使用Android Studio重新构建APK
2. 使用厂商提供的签名文件对APK进行系统签名
3. 将签名后的APK安装到系统分区
4. 应用将获得系统权限（uid=1000）

**需要**：
- 厂商的平台密钥（platform key）
- 或root权限手动安装到系统分区

### 方案C：Magisk模块（高级用户）
- **可行性**：中
- **复杂度**：高
- **效果**：需要测试

**操作步骤**：
1. 安装Magisk到车机（如果还没有）
2. 创建自定义Magisk模块
3. 修改AudioPolicyManager的逻辑或添加白名单
4. 重启车机

**风险**：
- 可能破坏系统稳定性
- 需要深入的Android系统知识
- 更新OTA可能失效

### 方案D：临时缓解措施
- **可行性**：低（只能部分缓解）
- **复杂度**：中
- **效果**：有限

**操作步骤**：
1. 创建系统启动脚本定期设置属性
2. 使用root权限手动设置：`su 0 setprop brlinkd.a2dp.muted 0`
3. 创建自动化工具定期检查和设置

**限制**：
- 无法解决AudioPolicyManager的UID检查
- 效果可能不持久

## 技术细节

### AudioPolicyManager的限制逻辑
```java
// AudioPolicyManager.getOutputForAttrInt() 中的检查逻辑
if (uid != 1000 && (flags & 0xA00) != 0) {
    // 第三方应用强制路由到扬声器
    selectedDeviceId = 0; // 扬声器
    return;
}

// 系统应用允许路由到A2DP
if (uid == 1000 && isA2dpConnected) {
    selectedDeviceId = a2dpDeviceId; // 蓝牙
    return;
}
```

### brlinkd.a2dp.muted 属性
- `0` = A2DP未静音
- `1` = A2DP静音
- **但是这个属性不影响AudioPolicyManager的UID检查**

## 结论

**最可靠的解决方案是联系车机厂商**，请求将应用加入音频白名单。这是：
- ✅ 安全的
- ✅ 持久的
- ✅ 不破坏系统稳定性
- ✅ 得到厂商支持的

技术层面的绕过方案（如修改系统文件、Magisk模块等）：
- ❌ 风险较高
- ❌ 可能破坏系统稳定性
- ❌ 难以维护
- ❌ OTA更新可能失效

建议优先联系车机厂商寻求官方支持。
